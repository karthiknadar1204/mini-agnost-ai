# SDK smoke test

`sdk-smoke-test.ts` tests the SDK the way a real app would — by importing the
**installed `logsneat` package**, not the local `../src` files. The SDK isn't
published, so you install it from a local tarball.

## Prerequisites

- The backend running: `cd apps/server && bun run dev` (port 3004)
- The detections worker running: `cd apps/server && bun run worker`
- Redis running (the worker needs it)
- A logsneat **API key** for a project — create one via the API:
  1. `POST /auth/signup` → copy the JWT
  2. `POST /v1/projects` (Bearer JWT) → copy the project `id`
  3. `POST /v1/projects/:id/api-keys` (Bearer JWT) → copy the full `key` (shown once)
- An `OPENAI_API_KEY`

## Run it

```bash
# 1. Pack the SDK into a tarball
cd packages/sdk
bun pm pack                       # -> logsneat-0.0.1.tgz

# 2. Create a clean consumer project OUTSIDE the monorepo
#    (so it resolves the tarball, not the workspace copy)
mkdir -p /tmp/logsneat-consumer && cd /tmp/logsneat-consumer
bun init -y
bun add /Users/<you>/.../packages/sdk/logsneat-0.0.1.tgz openai

# 3. Copy the test in and run it
cp /Users/<you>/.../packages/sdk/test/sdk-smoke-test.ts .
API_KEY=ln_… OPENAI_API_KEY=sk-… bun run sdk-smoke-test.ts
```

## Expected result

The script prints a real LLM answer and `expected failure caught: card declined`.
On the backend you should then see (Bearer JWT + `x-project-id: <projectId>`):

- `GET /v1/traces` → two traces: `handle_request` (ok) and `risky_request` (error)
- `GET /v1/traces/:traceId` for `handle_request` → tree:
  `handle_request [WORKFLOW] → get_city [TOOL]` + `OpenAI Chat Completions [LLM]`
- `GET /v1/stats/overview` → 2 traces, 2 tool calls, successRate 0.5
- `GET /v1/detections` → `trace_error` + `tool_error` (from the failing trace)

> Note: the `.tgz` is a build artifact — don't commit it (it's regenerated by
> `bun pm pack`).
